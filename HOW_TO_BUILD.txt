PREREQUISITES
─────────────
1. Android Studio (any recent version — Hedgehog 2023.1+ recommended)
   Download: https://developer.android.com/studio

2. Android SDK (installed automatically by Android Studio)

3. A physical Android phone (API 26 / Android 8.0 or higher) for
   testing, OR the Android emulator with a virtual camera enabled.


STEP-BY-STEP BUILD
──────────────────
1. Open Android Studio.

2. Choose "Open" (not "New Project") and select this folder:
        TimelapsApp/
   Android Studio will detect the Gradle build files automatically.

3. Wait for the Gradle sync to complete (first time downloads
   dependencies — requires internet; may take 1-3 minutes).

4. Connect your Android phone via USB with Developer Options +
   USB Debugging enabled, OR start an emulator.

5. Click the green ▶ Run button (or press Shift+F10).
   The app installs and launches on your device.

6. To build a standalone APK:
   Build menu → "Build Bundle(s) / APK(s)" → "Build APK(s)"
   The debug APK appears at:
       app/build/outputs/apk/debug/app-debug.apk


WHAT THE APP DOES
─────────────────
• Full-screen camera preview (portrait and landscape both work).
• Tap the ⚙ Settings button (top-left) to configure:
    - Capture interval: 1s / 2s / 5s / 10s / 15s / 30s / 1m
    - Camera: Back or Front
    - Overlay: None / Timestamp / Stopwatch / Custom text
    - Text size (slider) and position (6 corner/edge options)
    - Output video FPS: 24 / 30 / 60
• Tap the red record button to start capturing frames.
• A live counter shows frame count + elapsed time.
• Tap the red STOP button when done.
• The app encodes all frames into an MP4 using the device's
  built-in H.264 hardware encoder (no external libraries).
• The finished video is saved to:
    Movies/TimelapsApp/TIMELAPSE_<timestamp>.mp4
  and is visible in your phone's Gallery / Photos app.


ARCHITECTURE NOTES
──────────────────
• Language:     Kotlin
• UI:           Jetpack Compose + Material 3
• Camera:       CameraX (ImageCapture use case)
• Video encode: Android MediaCodec + MediaMuxer (NV12 / H.264)
• No external encoding libraries — lightweight APK


TROUBLESHOOTING
───────────────
• "Gradle sync failed" — make sure you have internet access the
  first time and that Android Studio's SDK is installed.

• "INSTALL_FAILED_USER_RESTRICTED" — enable USB Debugging on
  your phone: Settings → Developer Options → USB Debugging.

• Front camera not working — some devices don't have a front
  camera. The selector will silently fail and keep the last
  working camera.

• Video appears green / corrupt — this is a rare device codec
  quirk. Try changing the output FPS in settings.

• Video saves but doesn't appear in Gallery — open your Gallery
  app and pull down to refresh, or reboot the phone.
